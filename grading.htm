General Guidelines for Grading

 

•          You will receive ½ the points possible for a program that does not compile. There are no exceptions to this rule so please make sure that the program you turn in compiles with no errors or warnings.

 

•          You will receive 0 points if its obvious that no work has been put into the program.

 

•          The program must pass the style checker with no errors. All the points will be given if it passes and 0 points will be given if it does not pass the style checker. Generally, style checker points are worth 10% of the grade for a given program.

 

•          Programs must pass all tests exactly. Example input and output are given for each program. If the program does not produce the exact output required then no points will be given for that test.

 

•          The requirements listed in the guidelines must be met completely. If the requirement is not met then no points will be given for that requirement. Full points will be given if the requirement is met.

 

•          Elegance is assignment dependent. Some assignment do not elegance points. Those that do have a list of the things that will be graded based on elegance. Please review these carefully. Partial credit will be given for elegance. Elegance includes but is not limited to the following

                        •          Program should reflect the students understanding of the programming language. This includes using the correct type of statements and data structure.

                        •          Solution should reflect a well-thought out algorithm and not a hacked together solution that was done the night before. This includes the efficiency of the solution.

                        •          No unnecessary code including duplicate code, commented out code that does nothing, etc.
